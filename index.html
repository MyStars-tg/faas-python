quickstart.py 
from mystars_faas import Client

client = Client(api_key="ms_live_…")

# 1 — quote the price (GRAM/TON or USDT)
quote = client.pricing.get(type="stars", quantity=500)

# 2 — check the recipient resolves
client.recipients.check(type="stars",
    recipient={"username": "@friend"})

# 3 — create the order & pay on-chain
order = client.orders.create(
    type="stars", quantity=500,
    recipient={"username": "@friend"},
    payment_currency="ton",
    callback_url="https://your.app/webhook")

print(order.payment.address)  # → delivered in ~30 s

Everything the fulfilment rail does, typed.

The SDK wraps the full public API surface — pricing, recipient checks, orders, products, webhooks — with idempotency and signature verification built in.

Pricing

Live quotes

GET /v1/pricing returns the all-in amount with quote freshness (quoted_at, valid_until) and the live USDT/TON rate. USDT quotes itemise the ~1% swap fee.

Safety

Recipient checks

POST /v1/recipients/check resolves the @username before you pay — including the already_subscribed guard for Premium gifts.

Orders

Self-describing orders

Stars (50 – 1,000,000) or Premium (3/6/12 months). One-hour payment window; expires_at is the single source of truth.

Webhooks

Terminal events

delivered, reversed, failed, expired — signed callbacks with verification helpers in the SDK.

Reversals

Auto-refund on-chain

Wrong amount, wrong memo, unresolvable handle — the payment reverses to the sending wallet automatically. reversal_tx carries the receipt.

Limits

Documented budgets

Per-minute request budget with RateLimit-* / Retry-After headers; pricing & recipient probes capped at 30 req/min.
